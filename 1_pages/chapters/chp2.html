Chapter 2
: Types and Variables

    • Data Types

      • Data types are divided into two main sets: value and reference. These are divided by primitive types (see table 2.1). Value type data is stored in the program stack and directly contains its value. The amount of memory that a value type data uses is fixed, and it is determined by the primitive type. The memory allocated for a value type data is released when the program exits its ragne (scope). Reference type data keep a reference (an address) in the stack that points to the dynamic memory (heap) where the actual value is stored. The size allocated for a reference in the heap could change dynamically during the program execution. The reference only points to an object of the same primitive type. An object is a special primitive data type. It is the parent of all types in the .NET Framework; it can takes values of any other type. One of the biggest characteristic of a reference data is that can be assigned to null. Like mentioned above, the data types are divided into primitive types. These are: integer, real floating-point, real floating-point with decimal precision, bolean, character, string, and object. Each data uses an amount of space in the memory, this space unit is bit.

    • Decimal Accuracy

      • Precision of the Real Types
        decimal: precision of 28 to 29 decimal places
        double: precision of 15 to 16 decimal digits
        float: precision of seven decimal places

    • Nullables Types

      • Property that allows value type data to lack a value (null). This means that a value type data is converted into a reference type data. Null means that the data is not pointing to an object in the heap. To declare a nullbable value use the ? operator after the type declaration. For example:
        int?
        long?
        Some properties and methods to use with a nullable:

        .Value: returns the stored value (can't be null)
        .HasValue: checks whether it has a value or not (null); returns flase for null.
        .GetValueOrDefault(): returns the stored value when it is not null, default value otherwise.

    • Escaping Sequences

      • The escape operator is /, and it is used in front of special text that cannot be printed on the console. For quoted strings preceded by @, the rules of the escape character do not apply because everything within the "" will be printed as is. The only character that needs to use an escape character is the the double-quote. The way to do this is by using double-quote twice in a row.
