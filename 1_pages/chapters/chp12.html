Chapter 12
: Exception Handling

    • Exceptions

      • In general, a component of a C# program is intended to do an specific task; it will have a functionality. As a rule of thumb, this component will do what it was created for OR throw an exception. In the second scenario, the component fails to complete the task that it was created for. Thus, an exception is thrown because the component is not intended to behave outside of its functionality. The exception thrown has the capacity to details what was wrong (why and where in some cases).

    • Exception Handling

      • When an exception is thrown, the .NET Framework is expected to do something. This "something" is what is called exception handling, nut it is not 100% automatically. The programer needs to set up the handling process in the code.

    • Using exceptions and catching them

      • When the programer knows of the usual scenarios where the code's component will fail prior to the implementation of the code, he can wrap up those scenarios (exceptions) and handling them. The way this is implemented is by a try-catch blocks. In the try block, the code's component is wrapped up; in the catch block the handling part. It is important to mention that an exception could be thrown dua a possible scenaro or scenarios. The programer could creat multiple catch blocks (handling processes) depending on the type of the exception.

    • try-finally

      • The try-finally blocks is a "more complete" handling process. In the case an exception is been thrown and a resource was used, this resource needs to be released. So, it could happen that in the catch block the resource was not released. In this case we have a problem! The finally block will be always executed! Thus, any released line should be in it!


        The important method in IDisposable interface is Dispose(). The main thing we need to know about the method is that it releases the resources of the class that implements it. In cases when resources are streams, readers or files releasing resources can be done using the Dispose() method from IDisposable interface, which calls their Close() method. This method closes them and releases their resources.

    • using

      • "using" is the shorter form of any try-finally segment that uses a resource.


        The types of exceptions are packed up in the class Exception. This class contains all types of exceptions. There are two main classes (sub-groups): ApplicationException and SystemException. The programer will defined the catch block based on the type of the exception. The .NET Framework gives a trace whenever an exception is thrown. If the exception is not well defined it will be more difficult to pint it down in the given trace.

    • Advantages of Using Exceptions

      • Separation of the Exception Handling Code

        With this the programer avoid the "spagheti" code where the not intended behavior of the code's component in handled in between the written code. The problem with the "spagheti" code basically is readability. In it the logic of the code is interrupted with the handling's logic.

      • Information of the problem

        Give information of the problem: what causes it, where and why) using the call-stack (trace).
